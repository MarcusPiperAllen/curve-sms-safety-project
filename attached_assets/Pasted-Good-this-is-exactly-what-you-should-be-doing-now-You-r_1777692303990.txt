2. SIGN-UP FLOW

What the user does:

* Enters phone number
* Checks consent box
* Clicks “Activate Alerts”

What the system does:

* Saves phone number in database (subscribers)
* Sends SMS:

You're now connected to CurveLink.
To report an issue, text:
REPORT [your message]
Example:
REPORT suspicious activity near lobby
Reply STOP to unsubscribe.

3. USER STATE: “CONNECTED RESIDENT”

Now the user is in the system.

They don’t use a dashboard.

👉 Their interface is SMS (text messaging)

⸻

📱 4. HOW THE USER USES THE SYSTEM

A. Receive Alerts

Admin sends message → user receives SMS:

ALERT: Maintenance issue in Building A. Avoid lobby area.

⸻

B. Send a Report

User texts your Twilio number:

REPORT suspicious person near entrance

What happens:

* Twilio receives message
* Sends it to your app (/sms)
* Your app:
    * Parses message
    * Saves it to reports table
    * Associates it with that phone number

⸻

C. Opt Out

User sends:

STOP

System:

* Removes or deactivates them
* They stop receiving messages

⸻
